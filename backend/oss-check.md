# 阿里云 OSS 公共读权限设置检查清单

## 步骤 1: 确认 Bucket 名称
- [ ] Bucket 名称：`cangjiexing`
- [ ] Region：`西南 1（成都）`

## 步骤 2: 检查读写权限设置

### 方法 A: 通过概览页面
1. 进入 https://oss.console.aliyun.com/
2. 点击 `cangjiexing` bucket
3. 在 **概览** 页面，找到 **读写权限** 部分
4. 确认显示为 **公共读** (不是"私有")
5. 如果不是，点击右侧的 **修改** 按钮

### 方法 B: 通过数据安全页面
1. 进入 `cangjiexing` bucket
2. 左侧菜单点击 **数据安全** → **读写权限**
3. 确认 ACL 设置为 **公共读 (public-read)**
4. 如果不是，点击 **设置** 或 **修改**

## 步骤 3: 确认文件存在
1. 点击左侧 **文件管理** → **文件列表**
2. 确认存在文件 `BANNER_fcd5e9cbe8.jpg`
3. 点击文件名查看详细信息
4. 确认文件的 **存储类型** 和 **状态** 正常

## 步骤 4: 等待生效
- ACL 修改后通常立即生效
- 但如果使用了 CDN，可能需要等待几分钟

## 测试方法
修改后，在浏览器中直接打开这个 URL：
```
https://cangjiexing.oss-cn-chengdu.aliyuncs.com/BANNER_fcd5e9cbe8.jpg
```

应该能看到图片，而不是 XML 错误。

## 常见问题

### 问题 1: 修改后还是 403
- 确认修改的是正确的 bucket
- 清除浏览器缓存后重试
- 等待 1-2 分钟再测试

### 问题 2: 找不到"读写权限"设置
- 确认你有管理员权限
- 检查是否在正确的 region（成都）

### 问题 3: 设置了公共读但还是无法访问
- 检查是否启用了 CDN 加速域名
- 如果使用了自定义域名，需要单独配置 CORS
